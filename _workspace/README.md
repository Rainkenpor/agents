# _workspace

Shared module that gives MCP servers file-system and git-repository capabilities.

**Stack:** Node.js built-ins (`fs`, `child_process`) · TypeScript · Winston

---

## Overview

`_workspace` provides two independent services:

| Service | Responsibility |
|---------|---------------|
| `FilesService` | Read, write, delete and move arbitrary files with automatic history tracking |
| `RepositoryService` | Clone, edit, commit, push/pull and close git repositories identified by UUID |

Both services are stateless classes — instantiate once per MCP module and share the instance across all tool handlers.

---

## Storage layout

```
data/
├── workspace/
│   ├── files-history.json   ← append-only log of every FilesService operation
│   └── repos.json           ← registry of all cloned repositories
└── repos/
    ├── <uuid-1>/            ← git working tree (branch A of repo X)
    ├── <uuid-2>/            ← git working tree (branch B of repo X)
    └── ...
```

Repositories are stored in UUID-named folders so multiple MCPs (or the same MCP) can work on the same remote repository with **different branches** without path collisions.

---

## Integration

### 1. Import the services

```typescript
// In your MCP's tools.ts or directly in a tool file:
import { FilesService, RepositoryService } from "../_workspace/index.ts";

// Instantiate once — safe to share across all tools in the module
export const files = new FilesService();
export const repos  = new RepositoryService();
```

### 2. Custom data directory (optional)

By default data lives in `<cwd>/data/workspace` and repos in `<cwd>/data/repos`.
Override via the `WorkspaceConfig` constructor argument:

```typescript
import type { WorkspaceConfig } from "../_workspace/index.ts";

const cfg: WorkspaceConfig = {
  dataDir:  "/custom/path/workspace",
  reposDir: "/custom/path/repos",
  historyLimit: 5_000, // keep last 5 000 history entries
};

export const files = new FilesService(cfg);
export const repos  = new RepositoryService(cfg);
```

## FilesService API

### `readFile(filePath, actor?)`
Reads a file from disk and appends a `read` entry to the history log.
Returns `{ content: string, history: FileHistoryEntry[] }`.

```typescript
const { content, history } = files.readFile("/abs/path/to/file.txt", "my-mcp");
```

### `writeFile(filePath, content, options?)`
Creates or overwrites a file (parent directories are created automatically).
Appends a `write` entry to the history log.

```typescript
files.writeFile("/abs/path/to/output.json", JSON.stringify(data), {
  message: "generated by my-tool",
  actor: "my-mcp",
});
```

### `deleteFile(filePath, options?)`
Deletes a file and appends a `delete` entry to the history log.

```typescript
files.deleteFile("/abs/path/to/old.txt", { actor: "my-mcp" });
```

### `move(fromPath, toPath, options?)`
Moves or renames a file / directory. Appends a `move` entry to the history log.

```typescript
files.move("/path/a.txt", "/path/b.txt", { actor: "my-mcp" });
```

### `createDirectory(dirPath, options?)`
Creates a directory (recursive). Appends a `mkdir` entry to the history log.

```typescript
files.createDirectory("/path/to/new-dir", { actor: "my-mcp" });
```

### `listDirectory(dirPath)`
Returns an array of entry names; directories are suffixed with `/`.

```typescript
const entries = files.listDirectory("/path/to/dir");
// → ["file.ts", "subdir/"]
```

### `getHistory(filePath?)`
Returns history entries, optionally filtered by path.

```typescript
const all    = files.getHistory();
const forOne = files.getHistory("/abs/path/to/file.txt");
```

## RepositoryService API

### `clone(url, branch, options?)`
Clones a remote repository at the specified branch into a new UUID folder.
If an identical `(url, branch)` clone already exists the existing entry is
returned instead of cloning again (pass `{ force: true }` to override).

```typescript
const entry = await repos.clone(
  "https://github.com/org/repo.git",
  "feature/my-branch",
  { label: "mi proyecto" },
);
console.log(entry.id);   // UUID used as folder name
console.log(entry.path); // absolute path to the working tree
```

### `readFile(repoId, filePath)`
Reads a file relative to the repository root.

```typescript
const content = repos.readFile(entry.id, "src/index.ts");
```

### `writeFile(repoId, filePath, content)`
Creates or overwrites a file relative to the repository root.

```typescript
repos.writeFile(entry.id, "src/new-feature.ts", "export const x = 1;");
```

### `deleteFile(repoId, filePath)`
Removes a file using `git rm` (falls back to `fs.unlinkSync` for untracked files).

```typescript
await repos.deleteFile(entry.id, "src/old-file.ts");
```

### `move(repoId, from, to)`
Moves/renames a path inside the repository using `git mv`.

```typescript
await repos.move(entry.id, "src/old-name.ts", "src/new-name.ts");
```

### `createDirectory(repoId, dirPath)`
Creates a directory and adds a `.gitkeep` placeholder so git tracks it.

```typescript
repos.createDirectory(entry.id, "src/new-module");
```

### `listDirectory(repoId, dirPath?)`
Lists entries at a path inside the repository (defaults to root).

```typescript
const entries = repos.listDirectory(entry.id, "src");
```

### `commit(repoId, message, options?)`
Stages all changes (`git add -A`) and creates a commit.

```typescript
const out = await repos.commit(entry.id, "feat: add new feature");
```

### `push(repoId)`
Pushes the current branch to `origin`.

```typescript
await repos.push(entry.id);
```

### `pull(repoId)`
Pulls the latest changes from `origin`.

```typescript
await repos.pull(entry.id);
```

### `status(repoId)`
Returns the output of `git status --short`.

```typescript
const status = await repos.status(entry.id);
```

### `log(repoId, limit?)`
Returns the last N commit messages (default: 10).

```typescript
const log = await repos.log(entry.id, 5);
```

### `close(repoId)`
Deletes the repository folder from disk **and** removes it from the registry.
Use this when the MCP is done with a repository.

```typescript
repos.close(entry.id);
```

### `listRepos()`
Returns all registry entries.

```typescript
const all = repos.listRepos();
```

### `getRepo(repoId)`
Returns a single registry entry or `null`.

```typescript
const entry = repos.getRepo("some-uuid");
```

## Full MCP tool example

```typescript
// agent-myapp/tools/repo.tool.ts
import z from "zod";
import { ok } from "../types";
import type { ToolDefinition } from "../types";
import { RepositoryService } from "../../_workspace/index.ts";

const repos = new RepositoryService();

export const repoTools: ToolDefinition[] = [
  {
    name: "repo_clone",
    description: "Clone a git repository at a specific branch",
    inputSchema: {
      url:    z.string().describe("Remote repository URL"),
      branch: z.string().describe("Branch to checkout"),
      label:  z.string().optional().describe("Human-readable label"),
    },
    handler: async ({ url, branch, label }) => {
      const entry = await repos.clone(url, branch, { label });
      return ok({ id: entry.id, path: entry.path, branch: entry.branch });
    },
  },
  {
    name: "repo_commit_and_push",
    description: "Stage all changes, commit, and push to origin",
    inputSchema: {
      repoId:  z.string().describe("Repository ID returned by repo_clone"),
      message: z.string().describe("Commit message"),
    },
    handler: async ({ repoId, message }) => {
      const commitOut = await repos.commit(repoId, message);
      const pushOut   = await repos.push(repoId);
      return ok({ commit: commitOut, push: pushOut });
    },
  },
  {
    name: "repo_close",
    description: "Delete a cloned repository from disk",
    inputSchema: {
      repoId: z.string().describe("Repository ID to close"),
    },
    handler: async ({ repoId }) => {
      repos.close(repoId);
      return ok({ closed: repoId });
    },
  },
];
```

## Logging

All operations are logged to `logs/workspace.log` using the same Winston format as the rest of the project:

```
2026-04-14 10:23:01 [INFO]: [repo] cloning https://github.com/org/repo.git@main → /data/repos/abc123
2026-04-14 10:23:15 [INFO]: [repo:abc123] commit: feat: add new feature
2026-04-14 10:23:16 [INFO]: [repo:abc123] push
2026-04-14 10:24:00 [INFO]: [files] write → /some/path/output.json
```
